Orchestrator

🎯 Coordinator

Owns the sprint backlog. Reads state, checks the human inbox, decomposes work into subtasks, and delegates. Never writes code — only plans and tracks. Enforces task lifecycle: proposed → ready → in_progress → review → done.

Specialist

📐 Architect

Designs system architecture, API contracts, database schemas, and module boundaries. Outputs structured specs — never application code. Every decision includes a one-line rationale.

Specialist

💻 Coder

Implements features following architect specs. Touches only project/src/ and project/migrations/. One feature per task — no scope creep.

Specialist

🧪 Tester

Writes unit, integration, and acceptance tests. Reports results as structured JSON. Never modifies application code — reports failures back to the coordinator.

Specialist

🔍 Reviewer

Reviews for correctness, security (OWASP Top 10), and standards compliance. Structured findings with severity levels. Does not modify files.

Specialist

🔎 Researcher

Evaluates technologies using weighted trade studies. Conducts market analysis. Maintains a Technology Radar (Adopt/Trial/Assess/Hold). Evidence-based recommendations with confidence levels.

Key Principle

🚧 Strict Boundaries

Each agent has a defined filesystem scope. The coder can't modify tests. The reviewer can't change code. The tester can't alter the backlog. This prevents cascading errors and makes behavior predictable.

📊 Structured Output

Every heartbeat ends with a JSON status summary — tasks completed, in-progress, blocked. Machine-readable for dashboarding.

🪙 Token Efficient

File references over pasted content. JSON over prose. Minimal agent invocations per cycle. Each heartbeat costs only what it needs.

🔒 Safe Outputs

Creates PRs but never pushes to main or merges. Every change goes through a labeled PR gate for traceability.

Resilience

Self-Healing Mechanisms

When something breaks, the system diagnoses and fixes itself. Pre-defined recovery paths for every predictable failure mode.

Design Principle Most failures in autonomous AI systems are predictable: state corruption, task deadlocks, stale context, spec drift. A self-healing system has recovery paths for all of them.

🔄 State File Recovery

If ai-state.json becomes malformed or a task is stuck in in_progress for multiple heartbeats, the coordinator detects the anomaly, resets the task, and logs the recovery.

Stale task age > 3 heartbeats

🧪 Test-Driven Rollback

When tests fail after implementation, the coordinator routes failure details back to the coder. If the fix fails twice, the task is reverted and re-queued with a redesign flag for the architect.

Test failure after implementation

📋 Spec-Code Reconciliation

The reviewer detects implementation drift from architect specifications. The system generates a reconciliation task comparing spec against code and produces a targeted fix list.

Review finding = "spec_drift"

🧹 Memory Compaction

As MEMORY.md grows, older entries become noise. The coordinator periodically summarizes old entries, preserving key learnings while keeping the context window focused.

MEMORY.md > token threshold

🔀 Conflict Resolution

When two agent outputs conflict, the coordinator mediates: compares timestamps, checks which aligns with the latest directive, and routes the losing side for rework.

Overlapping file modifications

🚨 Deadlock Detection

If no task progresses for consecutive heartbeats, the coordinator diagnoses the root cause (circular dependency, unclear criteria) and redecomposes or escalates.

Zero progress for 3+ heartbeats

Agentic Programming Patterns

Key Concepts Explored

This project is a working laboratory for modern agentic programming patterns. Each concept is implemented and tested in the running system.

Concept What It Means How It's Implemented
Closed-Loop Autonomy The system operates end-to-end without human approval gates Auto-login → boot script → Docker → VS Code → agent wake. 18 auto-approve settings. Zero buttons.
Persistent Memory Agents retain knowledge across invocations without conversation history MEMORY.md for knowledge, ai-state.json for state, BOOT.md for context restoration
Agent Specialization Each agent has a narrow role with filesystem-scoped permissions 5 agents: coordinator plans, architect designs, coder implements, tester validates, reviewer audits
Self-Healing Recovery Pre-defined recovery paths for predictable failure modes State repair, test rollback, spec reconciliation, deadlock breaking, memory compaction
Structured Handoffs Agents communicate through typed JSON contracts, not free-form text Every output: task_id, files_modified, verdict, findings — deterministic routing
Async Human Control Humans steer without being in the loop for every decision INBOX.md with priority levels (critical/high/normal/low) checked every heartbeat
Electric Shock Protocol Full context restoration from cold start in a single file read BOOT.md contains project identity, sprint state, tool inventory, and action protocol
Token Efficiency Minimize LLM consumption through references and structured output File references via #file:path, JSON responses, minimal agent invocations per cycle
